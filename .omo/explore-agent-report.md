## 3. main.rs — Complete Contents

**File:** `C:\Users\pauma\Documents\playground\FuseBox\main.rs`
**Size:** 14,880 bytes | **Lines:** 361 (as read) / 324 (per PowerShell count, the difference likely being blank-line counting heuristics)

This is a **Rust async sandbox supervisor** (a "Process API") implementing:

- **PID 1 zombie reaper** — runs `waitpid(WNOHANG)` in a loop when running as PID 1 (inside Firecracker VM)
- **OOM guard daemon** — polls cgroup `/sys/fs/cgroup/memory/process_api/` directories and SIGKILLs processes exceeding memory limits
- **Control API** (port 2025) — Hyper HTTP server with `/status` and `/mount_root` endpoints
- **WebSocket gateway** (port 2024) — accepts WS connections, spawns `bash` subprocesses per connection, pipes stdin/stdout/stderr through WebSocket JSON messages (`{"stream":"stdout","text":"..."}`)
- **cgroup enrollment** — enrols subprocess PIDs into per-UUID cgroups with optional memory limits
- **System mount** — reads `/mount_config.json` and writes readiness sentinel to `/tmp/rclone-mounts/ready`

Key crates used: `tokio`, `hyper`, `tokio-tungstenite`, `clap`, `nix`, `uuid`, `anyhow`, `futures-util`, `http-body-util`, `tracing`.
